# 开发流程

本项目采用“每个任务一个分支、一个 PR”的开发流程。

## Claude 执行规则

Claude 不能直接在 `main` 分支上实现功能。

每个任务都必须按下面流程执行：

1. 从最新的 `main` 创建新分支。
2. 每个分支只处理一个边界清晰的任务。
3. 完成后运行相关测试和检查。
4. 提交本次任务代码。
5. 创建 PR 等待审核。
6. PR 未审核通过前不能合并。

## 分支命名

分支名要简短、清楚，例如：

- `feat/backend-scaffold`
- `feat/api-keys`
- `feat/mail-fetch-graph`
- `fix/oauth-refresh-errors`
- `test/mail-account-claim`

## PR 要求

每个 PR 需要说明：

- 改了什么。
- 为什么要这样改。
- 做了哪些测试。
- 是否涉及数据库迁移或部署变化。
- 如果是前端改动，必要时附截图。

## 回退策略

因为每个任务都通过独立 PR 合并，如果某次改动出问题，可以直接 revert 对应 PR 的 merge commit。

数据库迁移需要特别谨慎。能兼容旧版本时，优先做向后兼容迁移；不能兼容时，PR 里必须写清楚回滚方式。

## main 分支

`main` 分支始终代表最近一次审核通过的稳定状态。
